# OOAD_SmartCampus_Project

# Smart Campus Facility Booking & Maintenance System

## Project Overview
[cite_start]This project is a comprehensive solution designed to automate and streamline the management of campus facilities at Universities[cite: 39, 47]. 
[cite_start]The system enables event organizers to book venues, allows facility managers to oversee allocations, and provides administrators with tools for system maintenance and reporting[cite: 54, 55, 65, 72].

[cite_start]Built using **Object-Oriented Analysis and Design (OOAD)** principles, the application adheres to the **MVC Architecture** and incorporates various **Design Patterns** to ensure scalability and maintainability[cite: 4, 27, 28].

---

## Key Features
[cite_start]The system supports three primary user roles, each with specific modules[cite: 53, 54]:

### 1. Event Organizer (Club Head)
* [cite_start]**Manage Event Booking:** Request venue slots and upload event details[cite: 58, 60, 64].
* [cite_start]**Booking Control:** Modify or cancel existing requests[cite: 61, 88].
* [cite_start]**Maintenance:** Inform or assign maintenance for venues[cite: 62, 90].

### 2. Facility Manager
* [cite_start]**Allocation Control:** Approve or reject incoming venue requests[cite: 68, 71, 89].
* [cite_start]**Rule Management:** Set and enforce specific venue rules[cite: 69, 91].
* [cite_start]**Monitoring:** Monitor facility usage and status[cite: 70, 128].

### 3. Administrator (System Manager)
* [cite_start]**User & Facility Management:** CRUD operations for system users and campus resources[cite: 74, 75, 99, 102].
* [cite_start]**Master Schedule:** Maintain a centralized schedule to prevent booking conflicts[cite: 76, 84, 108].
* [cite_start]**System Health:** Generate reports and manage database backups[cite: 77, 78, 111, 113].
